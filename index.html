X65

Modern 8-bit microcomputer

READY
█

X65 is an effort to build an 8-bit microcomputer for the modern era. It uses best of breed components and strives to keep 8-bit feeling, while being usable for daily basis computing activities.

Components

  • CPU: 65816, 40MHz
  • RAM: 16MB
  • Video: VIC-IV, DVI-I
  • Sound: YMF278, Stereo RCA, S/PDIF
  • Coprocessor: J1, 2x CPU clock
  • Input/Output:
    • 2x 8520 CIA
    • Prioritised interrupts controller
  • Ports:
    • 2x SD Card slot
    • 2x DB9 Joystick port
    • 4x USB port
    • 2x MIDI/Serial
    • Ethernet 1Gb RJ45 port
    • 8-bit ISA expansion port
    • Clock port
  • Keyboard: 60% form-factor, mechanical, programmable layout
  • Power supply: 12V DC power brick

Few of mentioned chips are borderline unobtainium nowadays and few others exist in HDL only, thus physical hardware is implemented in Intel CycloneÂ® V SE 5CSEBA6U23I7 FPGA chip. Design and prototype is based on MiSTer project.

Please mind, that specifications are not set in stone and will change during development. Interfaces for modern hardware ports are still in flux. You are welcome to challenge them, to make X65 even better. See contact details below.

Software

Every new machine faces a lack of software, so it is best to tap into existing software library. For this reason X65 is fairly compatible with Commodore 64. It is not 100% backward compatible though and strives to keep the porting effort on a similiar level to porting C64 software for IDE64.

Operating System

On Power-On machine starts WingsOS from (flashable) ROM.

Why?

Because we need "Understandable Computers" to make computers unstandable.

Raspberry Pi is a step in right direction, but it is still a computer with layers and layers of abstraction separating the human from underlying machine.

F.A.Q.

  • 65816 is not 8-bit!
    65_8/16 is a hybrid microprocessor allowing one to write software the same way as for 6502, but with a lot more memory available and ability to compute using 16bit numbers when needed.
  • Why no SID?
    While understanding the sentiment, SID is not "best of breed". J1 subroutine will provide SID emulation on OPL3, when running C64 software.
  • Why no DMA?
    J1 is powerfull and flexible enough to serve as DMA, HDMA, display-list, blitter, etc.
